# Security Policy

## Supported Scope

MPx Player is an actively developed project, and security reports are welcome for issues that affect:

- local file access and exposure
- downloader behavior and execution paths
- platform bridge misuse
- package or dependency risk with real application impact
- release build configuration that weakens app safety

## Reporting a Vulnerability

Please do not open public issues for undisclosed security vulnerabilities.

Instead, report the issue privately to the project maintainer with:

- a clear description of the problem
- affected platform and version
- reproduction steps
- proof of concept if safe to share
- impact assessment

If no dedicated security contact exists yet, use the repository owner contact route and clearly label the message as a security report.

## What to Expect

We will aim to:

- acknowledge the report
- validate the issue
- assess severity and affected scope
- prepare a fix or mitigation
- disclose responsibly once users can protect themselves

## Disclosure Guidance

Please allow reasonable time for investigation and remediation before public disclosure.

Responsible reporting helps protect users and contributors working on the project.
